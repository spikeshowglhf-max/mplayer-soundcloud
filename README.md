# Mplayer SoundCloud

Статический кастомный музыкальный плеер для GitHub Pages, который использует публичные ссылки SoundCloud, официальный widget API и `oEmbed` для метаданных.

## Desktop app

Проект теперь можно собрать и как полноценное Windows-приложение на Electron.

Команды:

```powershell
npm.cmd install
npm.cmd start
npm.cmd run dist:portable
```

Готовый `.exe` после сборки появится в папке `dist-desktop`.

## Android APK

Проект можно собрать и как Android-приложение через Capacitor.

Локальные команды:

```powershell
npm.cmd install
npm.cmd run android:add
npm.cmd run cap:sync
```

Для автоматической сборки APK в репозитории есть GitHub Actions workflow.

## Что уже есть

- Кастомный интерфейс поверх SoundCloud widget
- Воспроизведение, пауза, следующий и предыдущий трек
- Прогресс-бар и перемотка
- Громкость и mute
- Shuffle и repeat
- Очередь с drag-and-drop сортировкой
- Импорт и экспорт очереди в JSON
- Сохранение очереди и настроек в `localStorage`
- Горячие клавиши: `Space`, `←`, `→`, `M`, `S`, `R`
- Настраиваемый акцентный цвет

## Как запустить локально

Для полного функционала лучше запускать через локальный сервер, потому что `fetch`, `service worker`, `clipboard` и часть мобильных возможностей в `file://` работают нестабильно.

Простой вариант:

```powershell
python -m http.server 8080
```

После этого открой `http://localhost:8080`.

## ПК и телефон

- На ПК включается двухколоночная раскладка: слева sticky-плеер, справа библиотека и очередь.
- На телефоне интерфейс перестраивается в мобильный режим с крупными touch-кнопками, нижним mini-player dock и безопасными отступами под экран.
- Очередь можно переставлять мышкой на ПК и кнопками `Up` и `Down` на телефоне.
- Добавлены `manifest.webmanifest`, `service worker` и `Media Session API`, так что сайт удобнее ставить на домашний экран и использовать на мобильных устройствах.
- Само приложение кэшируется локально, но музыка из SoundCloud все равно требует интернет.

## Как выложить на GitHub Pages

1. Создай репозиторий на GitHub.
2. Загрузи в него файлы этого проекта.
3. Открой `Settings` -> `Pages`.
4. В `Build and deployment` выбери `Deploy from a branch`.
5. Выбери ветку `main` и папку `/ (root)`.
6. Сохрани настройки и подожди публикацию.

После этого сайт будет доступен по адресу GitHub Pages твоего репозитория.

## Важные ограничения

- GitHub Pages хостит только статические файлы, поэтому здесь нет серверной части.
- Плеер работает с публичными ссылками SoundCloud. Приватные, удаленные, регионально ограниченные или запрещенные для embed материалы могут не открыться.
- Воспроизведение зависит от правил браузера: autoplay может быть ограничен до первого взаимодействия пользователя.
- Фоновое воспроизведение и поведение на заблокированном экране зависят от мобильного браузера, iOS/Android и самого SoundCloud widget.
- Метаданные берутся через `https://soundcloud.com/oembed`, а управление воспроизведением идет через `https://w.soundcloud.com/player/api.js`.
